SoundFacets

An auditory add-on for the Exhibit 3.0 faceted browser

Demo: "Some Sound Observations" by Pauline Oliveros

Setup

0: Setup Exhibit

See "Getting Started with Exhibit 3.0". If you are not hosting your own installation of Exhibit, be sure to link to API version 3.1.1 or higher, as below:

<script src="http://api.simile-widgets.org/exhibit/3.1.1/exhibit-api.js"></script>

1: Include script

Copy and paste the following line into the head of the HTML document where you run Exhibit:

<script src="http://web.mit.edu/akstuhl/www/soundfacets/soundfacets.js"></script>

2: Edit JSON

Use the following syntax to denote sounds you wish to associate with items in your data set. See oliveros.js for an example.

"sound-example": "on",

3: Add sound files

Copy .mp3 sound files for each of the associated sounds into a directory called "sounds". These files should have the same names as used in your JSON file, e.g. "sound-example.mp3".

4: Add sound facets

Add facets whose "ex:expression" values match the same naming convention, as in the example below. The facet label can be whatever text you want to display.

<td><div ex:role="facet" ex:expression=".sound-example" ex:facetLabel="Example"></div></td>

5: Include CSS

Copy and paste the following line into the head of your html document:

<link href="http://web.mit.edu/akstuhl/www/soundfacets/soundfacets.css" type="css" rel="stylesheet" />

Instead or in addition, you can style sound facets by targeting .exhibit-facet[ex\:expression*='.sound-'] in your custom CSS

Technical Notes

SoundFacets requires support for both Exhibit 3.0 and the WebAudio API. As of writing, SoundFacets works correctly in Firefox 34.0.5, Safari 7.1.2, and Chrome 39.0.2171.95.

Troubleshooting

SoundFacets is still in initial development. If you encounter an issue or have any questions, please contact akstuhl@mit.edu.
